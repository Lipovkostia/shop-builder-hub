## Проблема

1. На «Главная страница — оформление» загрузка фонового изображения и правых блоков молча «проходит», но картинка не появляется. Нет ни клиентской валидации размера/типа, ни понятной ошибки — при отказе (например, слишком большой файл, неподдерживаемый формат, HEIC с айфона) пользователь видит только «загружено».
2. Ни в одном окне загрузки на сайте не указаны реальные лимиты (размер файла, форматы, рекомендуемые размеры). Пользователь не понимает, какой файл подсунуть.

## Реальные лимиты бакетов (из БД)

| Бакет | Где используется | Макс. размер | Разрешённые форматы |
|---|---|---|---|
| `product-images` | Фото товаров, каталог, Авито | **100 МБ** | JPEG, PNG, WEBP, GIF |
| `product-videos` | Видео карточки товара | **500 МБ** | MP4, WEBM, MOV |
| `landing-info` | Hero главной, правые блоки, инфо-блоки | **100 МБ** | любые изображения |
| `landing-slides` | Слайдер главной | **100 МБ** | любые изображения |
| `avito-images` | Фото объявлений Авито | **100 МБ** | любые изображения |
| `order-attachments` | Вложения к заказам | **100 МБ** | любые файлы |
| `ai-history` | История AI (внутр.) | **100 МБ** | — |

Рекомендуемые размеры (для UI-подсказок):
- Hero-баннер: **1920×900 px**, JPG/WEBP, до 2 МБ
- Правые блоки главной: **800×600 px**, JPG/WEBP, до 1 МБ
- Слайды: **1920×800 px**, JPG/WEBP, до 2 МБ
- Инфо-блоки: **800×600 px**, до 1 МБ
- Товары: **1000×1000 px**, JPG/WEBP, до 3 МБ (на карточку)
- Авито: **1280×960 px**, JPG, до 5 МБ (лимит Авито 25 МБ)
- Логотип/иконка в шапке: **512×512 px**, PNG, до 500 КБ

## План

### 1. Общий компонент подсказки
Создать `src/components/admin/UploadHint.tsx` — маленькая строка серым текстом:
`Формат: JPG/PNG/WEBP · Реком.: 1920×900 · До 2 МБ`

### 2. Общая клиентская валидация
Добавить утилиту `src/lib/uploadValidation.ts`:
- `validateImageFile(file, { maxBytes, mimes, recommend })` — возвращает `{ ok, error }`.
- Проверяет `file.size`, `file.type` (с фолбэком на расширение для HEIC/др.), выдаёт человекочитаемую ошибку через toast: «Файл 12 МБ — слишком большой, максимум 2 МБ» / «Формат HEIC не поддерживается, сохраните как JPG».

### 3. Починить загрузку hero
В `src/components/admin/HomepageHeroManager.tsx`:
- Вызывать `validateImageFile` до `storage.upload`.
- Проверять фактическую подписку через `data.path` и сразу подставлять URL; если `getPublicUrl` вернул пусто — показывать ошибку.
- Логировать любые ошибки бакета в toast (сейчас есть, но валидация отсечёт частый случай).
- Показать под превью подсказку `UploadHint` (лимит + рекомендуемый размер отдельный для hero и для правых блоков).

### 4. Добавить подсказки во все точки загрузки
Пройти по всем местам и вставить `UploadHint` + вызов `validateImageFile`:
- `HomepageHeroManager.tsx` (hero + side blocks)
- `SlidesManager` (слайды главной)
- `LandingInfoBlocksManager` (инфо-блоки)
- `RetailPartnersManager` (партнёры)
- `ImageGalleryViewer.tsx` (галерея товара, «+»)
- `StoreFront.tsx` (drag&drop фото товара)
- `AvitoSection.tsx` / `avito-image-edit` UI (фото Авито)
- Настройки магазина: логотип, favicon (если есть форма)
- Форма отзыва / вложений к заказу (order-attachments), если UI существует

Для каждого блока — свой пресет лимитов из таблицы выше.

### 5. Проверить
- Загрузить в hero JPG 500 КБ → появится превью и сохранится.
- Загрузить PNG 30 МБ → toast «Слишком большой, максимум 2 МБ», upload не идёт.
- Загрузить HEIC → toast с рекомендацией сохранить как JPG.
- Убедиться, что во всех окнах загрузки под кнопкой видна строка с лимитами.

## Технические детали

- Валидацию делаем клиентски, потому что серверные лимиты в 100 МБ слишком грубые.
- `UploadHint` — чистый presentational компонент, без логики.
- `uploadValidation.ts` возвращает ошибку строкой; вызывающий сам показывает toast, чтобы сохранить контекст.
- Ничего в edge functions и БД не трогаем.
