
## Что делаем

Полностью переделываем главную страницу `/` в формат витрины интернет-магазина (без цен). Старую версию сохраняем и даём возможность переключаться между «новой» и «старой» главной прямо из админки.

## Новая главная страница

Структура (десктоп — 3 колонки, мобайл — стек: поиск → товары → категории → партнёры):

```text
┌──────────────────────────────────────────────────────────────┐
│  [ЛОГО]   ⌕  Большая строка поиска товаров…           [Вход]│
├──────────┬───────────────────────────────────┬───────────────┤
│ Каталог  │   Витрина товаров                 │ Где купить    │
│          │   ┌────┐ ┌────┐ ┌────┐ ┌────┐     │ в розницу     │
│ • Сыры   │   │ 📷 │ │ 📷 │ │ 📷 │ │ 📷 │     │ Наши партнёры │
│   – Тв.  │   │имя │ │имя │ │имя │ │имя │     │               │
│   – Мяг. │   └────┘ └────┘ └────┘ └────┘     │ ┌───────────┐ │
│ • Колб.  │   ┌────┐ ┌────┐ ┌────┐ ┌────┐     │ │  Баннер 1 │ │
│ • Масло  │   │ 📷 │ │ 📷 │ │ 📷 │ │ 📷 │     │ └───────────┘ │
│ • …      │   └────┘ └────┘ └────┘ └────┘     │ ┌───────────┐ │
│          │                                   │ │  Баннер 2 │ │
│          │                                   │ └───────────┘ │
└──────────┴───────────────────────────────────┴───────────────┘
```

Карточка товара показывает **только фото и название** (категория мелким текстом). Цены, SKU, количества — скрыты. Клик по карточке открывает крупный просмотр фото и описания (без цены, без корзины), внизу — кнопка «Где купить» со скроллом к блоку партнёров.

Поиск: моментальная фильтрация по названию + категориям. Если ничего не найдено — мягкое пустое состояние.

Левая колонка (Каталог):
- Иерархия категорий (parent → children) из тех же данных, что отдаёт `landing-products`.
- Клик по категории фильтрует витрину.
- Сворачивающиеся секции, активная подсвечена.

Правая колонка (Где купить в розницу. Наши партнёры):
- Заголовок «Где купить в розницу» + подзаголовок «Наши партнёры».
- Список карточек-баннеров: каждая = картинка + название + ссылка на сайт партнёра (открывается в новой вкладке, `rel="noopener"`).
- Если партнёров нет — блок аккуратно скрывается.

Авторизация/CTA: маленькая кнопка «Вход» в шапке ведёт на старую форму логина (без её удаления).

## Сохранение старой версии + переключатель

- Текущий `src/pages/Index.tsx` переименовываем в `src/pages/IndexLegacy.tsx` (содержимое без изменений).
- Создаём новый `src/pages/Index.tsx` — обёртка, которая читает флаг `homepage_version` из таблицы `landing_settings` и рендерит либо `IndexNew`, либо `IndexLegacy`. Пока флаг грузится — лёгкий лоадер, чтобы не было мигания. Значение по умолчанию — **новая** версия.
- Сам новый дизайн живёт в `src/pages/IndexNew.tsx` + компоненты `src/components/landing-shop/*` (Header, CatalogSidebar, ProductsGrid, ProductCard, PartnersSidebar, ProductQuickView).

В админке (`AdminPanel` → раздел «Главная страница» / Landing):
- Переключатель «Версия главной страницы»: `Новая витрина` / `Старая (классическая)`. Сохраняется в `landing_settings.homepage_version`.
- Подраздел «Партнёры розницы»: таблица с CRUD (название, ссылка, изображение-баннер, порядок, активность). Drag-and-drop сортировка.

## Технические детали

База данных (одна миграция):
- `ALTER TABLE landing_settings ADD COLUMN homepage_version text NOT NULL DEFAULT 'new'` (значения `'new' | 'legacy'`).
- Новая таблица `public.landing_retail_partners`:
  - `id uuid pk`, `name text`, `url text`, `image_url text`, `sort_order int`, `is_active bool default true`, `created_at`, `updated_at`.
  - `GRANT SELECT ON public.landing_retail_partners TO anon, authenticated;`
  - `GRANT ALL ON public.landing_retail_partners TO service_role;`
  - `GRANT INSERT, UPDATE, DELETE ON public.landing_retail_partners TO authenticated;`
  - RLS: публичное чтение активных; запись — только пользователям с ролью `super_admin` через существующую `has_role`.

Edge function:
- Расширяем существующий `landing-products`, чтобы он же отдавал `partners` (active, отсортированные) и `homepage_version` — одним вызовом, без лишних round-trip.

Картинки партнёров грузим в существующий стораджевый бакет с публичным чтением (тот же, что для лендинга/ритейл-баннеров — выберу по факту, без новой бакет-настройки, если подходит).

Изображения товаров берём из уже существующего ответа `landing-products` (поле `image`, `images_count`). Карточка лениво подгружает картинку (`loading="lazy"`).

SEO: один `<h1>` «Наш ассортимент» (визуально скрытый, чтобы дизайн остался чистым), `<title>` и meta description обновляем под витрину; добавляем JSON-LD `ItemList`.

## Чего НЕ делаем (явно)

- Не трогаем розничные/оптовые витрины магазинов (`/retail/*`, `/wholesale/*`) — у них своя логика.
- Не показываем цены, корзину, кнопки «купить» на новой главной. Корзина-демо со старой главной не переносится.
- Не удаляем старый код Index — он переезжает целиком в `IndexLegacy.tsx` и доступен переключателем.
