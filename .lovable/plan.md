## План исправления панели «Массовые значения»

### 1. Убрать причину обрезания
Сейчас панель открывается справа, но её содержимое не ограничено реальной высотой экрана: нижние действия и кнопки у полей оказываются ниже видимой области. Я перестрою Sheet как фиксированный рабочий контейнер:

```text
┌──────────────────────────────┐
│ Заголовок + выбранные + X     │  sticky top
├──────────────────────────────┤
│ Быстрые действия              │  всегда сверху
├──────────────────────────────┤
│ Прокручиваемая зона полей     │  единственная область скролла
│ Адрес                         │
│ Категория                     │
│ Цена / Promo / CPC            │
│ Контакты                      │
│ API / Настройки               │
├──────────────────────────────┤
│ Excel · ZIP · XML · Сохранить │  sticky bottom, всегда видно
└──────────────────────────────┘
```

### 2. Сделать панель строго в пределах экрана
- `SheetContent`: `h-[100dvh]`, `max-h-[100dvh]`, `overflow-hidden`.
- Ширина: не больше экрана — `w-[min(520px,100vw)]`, на узких экранах `w-screen`.
- Добавить нижний safe-area отступ, чтобы macOS Dock/браузерная нижняя зона не перекрывали кнопки.
- Убрать любые внутренние элементы, которые могут растягивать панель шире/ниже контейнера.

### 3. Перенести критичные кнопки в закреплённый низ
Видимые всегда:
- `Скачать Excel`
- `Скачать ZIP с фото`
- `XML ссылка`
- `Сохранить настройки`

Они будут в нижней sticky-панели, а не в самом конце длинного списка.

### 4. Переделать строки полей, чтобы кнопки справа не обрезались
Сейчас у каждого поля справа маленькие кнопки применения. Они видны частично, потому что строка не умеет адаптироваться. Сделаю единый компактный шаблон:

```text
[ поле / select на всю ширину ]
[ ✓ ко всем ] [ ✓ к выбранным ]
```

На широком месте можно оставить в одну строку, но при нехватке ширины кнопки переносятся вниз и остаются полностью кликабельными.

### 5. Сгруппировать длинную форму
Чтобы не была «портянка», разделю на понятные блоки с `Collapsible`:
- Основное: адрес, категория, вид объявления, вид товара
- Цена и продвижение: источник цены, Promo, CPC
- Контакты: лицо, телефон, почта, компания, аудитория
- API и настройки

Первый блок и «Цена/продвижение» будут доступны сразу; остальные раскрываются по необходимости.

### 6. Проверка результата
После реализации проверю именно текущий сценарий со скриншота:
- viewport около `1189x800`
- панель справа открыта
- нижние кнопки видны без прокрутки страницы
- кнопки применения у полей не уходят за правый край
- список внутри панели прокручивается, а не вся панель целиком
