## Проблема

Сейчас раздел Avito состоит из 3 колонок в одной строке: **Категории · Массовые значения (форма) · Таблица**. Все три фиксированной ширины — суммарно шире экрана. Средняя колонка с формами «прячется» под таблицей: кнопки справа от полей (✓ применить ко всем / к выбранным) и нижние кнопки (Экспорт, Скачать Excel, Скачать ZIP, XML-фид) обрезаны.

Увеличение `bulkWidth` не решает проблему системно — на ноутбуке всегда не хватит ширины для одновременного показа трёх плотных колонок + столбцов таблицы.

## Цель

Сделать так, чтобы при любой ширине экрана:
- Таблица занимала максимум полезного пространства и не «съедала» соседей.
- Форма массовых значений была полностью видна, когда нужна, и убиралась, когда не нужна.
- Все кнопки (применить, экспорт, API, настройки) были всегда доступны и не обрезались.
- Категории слева остались для быстрой навигации.

## Новая модель: Sheet-drawer вместо третьей колонки

Главное решение — **убрать «Массовые значения» из ряда колонок** и вынести в выезжающую панель (Sheet) поверх таблицы.

```text
┌──────────────────────────────────────────────────────────────────┐
│  Категории │  Таблица товаров (на всю оставшуюся ширину)         │
│   (свор.)  │  ┌──────────────────────────────────────────────┐   │
│            │  │ Tool-bar: [Поиск] [Цена▾] [⚙ Массовые …] [⋯] │   │
│            │  ├──────────────────────────────────────────────┤   │
│            │  │ Группа | Публ | Статус | Фото | Название …  │   │
│            │  │ …                                            │   │
└──────────────────────────────────────────────────────────────────┘

При нажатии на «Массовые значения» / при выборе строк галочкой —
выезжает Sheet справа (480-560 px), оверлей с затемнением:

                                   ┌─────────────────────────────┐
                                   │ Массовые значения        ✕  │
                                   ├─────────────────────────────┤
                                   │ ▸ Применить ВСЁ             │
                                   │   [К выбранным] [Ко всем]   │
                                   │                             │
                                   │ Аккордеоны:                 │
                                   │  ▾ Контакты                 │
                                   │     Имя, Тел, Почта, …      │
                                   │  ▸ Объявление               │
                                   │  ▸ Цена и продвижение       │
                                   │  ▸ Авито-категория          │
                                   │                             │
                                   ├─────────────────────────────┤
                                   │ [API] [Настройки] [Экспорт▾]│
                                   └─────────────────────────────┘
```

### Что меняется

1. **Топ-бар над таблицей** (новый):
   - Поиск (название/артикул/ID) — переезжает сюда.
   - Фильтр цены (Все / =0 / >0).
   - Счётчик «Показано: N из M».
   - Кнопка **«Массовые значения»** (с бейджем кол-ва выбранных) — открывает Sheet.
   - Меню «⋯»: API, Настройки, Экспорт (Excel/ZIP/XML-ссылка).

2. **Sheet «Массовые значения»** (Radix Sheet справа):
   - Ширина 480 px на десктопе, 100% на мобильном.
   - Sticky-шапка с заголовком и крестиком.
   - Sticky-нижняя панель с финальными действиями (Экспорт-меню, API, Настройки) — всегда видна, не обрезается.
   - Содержимое в `ScrollArea` + **Accordion**-секциях: Контакты · Объявление · Цена и продвижение · Категория Avito. По умолчанию открыта первая. Это решает «портянку из 15 полей».
   - В каждом поле кнопки `✓ ко всем` / `✓ к N выбранным` остаются справа от инпута, но ширина инпута стабильная (флекс 1) — никакого обрезания.
   - Sheet **не-модальный** (`modal={false}`) — пользователь может одновременно тыкать в таблицу (отмечать строки), а формы остаются открытыми и реагируют на изменение `selectedFeedProducts`.

3. **Левая колонка «Мои категории»** — без изменений, остаётся резизабельной и сворачивающейся в узкий рельс (уже реализовано).

4. **Таблица** — занимает 100% оставшейся ширины. Уже виртуализированная, столбцы резизабельные — просто получит больше места.

5. **Авто-открытие Sheet** при первом массовом выборе (галочки в таблице), чтобы не «терять» функционал. Управляется флагом в localStorage, чтобы не назойливо.

## Технические детали

**Файл:** `src/components/admin/AvitoSection.tsx`

- Удалить блок `bulkCollapsed ? рельс : <div … bulkWidth …>` (строки ~2719-…end-bulk-panel) из ряда колонок.
- Удалить состояние `bulkWidth`, ресайзер `onBulkResizeStart`, `bulkResizingRef`, `localStorage avito_bulk_width` (не нужны).
- Оставить `bulkCollapsed` → переименовать в `bulkSheetOpen` (инверсно).
- Перенести содержимое (Поиск, Фильтр цены, Массовые значения, Экспорт-блок, API, Настройки) в новый компонент `<AvitoBulkSheet>` в том же файле или рядом, обёрнутый в `Sheet` из `@/components/ui/sheet` со `side="right"`, `modal={false}`.
- Структура Sheet:
  - `SheetHeader` (sticky top) — заголовок + бейдж выбранных.
  - `ScrollArea` (flex-1) — `Accordion type="multiple" defaultValue={["bulk-apply","contacts"]}`:
    - `bulk-apply` — мастер-кнопка «Применить ВСЁ».
    - `contacts` — Контактное лицо, Телефон, Почта, Компания, Аудитория.
    - `ad` — Адрес, Категория Авито, Тип товара, Подтип.
    - `pricing` — Источник цены, Ставка CPC, Промо (Manual/Auto + поля).
  - `SheetFooter` (sticky bottom) — кнопки: `[Скачать Excel] [ZIP] [XML ссылка] · [API ▸] [Настройки ▸]` (или dropdown «Экспорт ▾» если узко).
- Поиск + фильтр цены вынести в новый `<AvitoToolbar>` над таблицей (компактный, h-9).
- Кнопка открытия Sheet: `<Button onClick={() => setBulkSheetOpen(true)}>Массовые значения {selectedCount > 0 && <Badge>{selectedCount}</Badge>}</Button>`.
- Auto-open: `useEffect` — когда `selectedFeedProducts.size` переходит с 0 на >0 и `localStorage.avito_bulk_autoopen !== "off"`, открыть Sheet.
- Меню «Экспорт» — `DropdownMenu` (по нашим правилам — `DropdownMenuGroup` + `onSelect`).
- Mobile (`< 768px`): Sheet `side="bottom"` с `max-h-[85vh]`.

**Что НЕ трогаем:**
- Логику `avitoFeed`, `bulkUpdateProductParams`, `saveDefaults` — никаких изменений.
- Таблицу, виртуализацию, колонки.
- Левый сайдбар категорий.
- Edge-функции и БД.

## Проверка после реализации

1. На 1189 px viewport (текущий) — таблица и форма не накладываются, все кнопки внутри Sheet видны (включая нижние Экспорт/API/Настройки).
2. Открыть Sheet → выбрать 3 строки → бейдж обновляется → кнопка «К выбранным (3)» активна.
3. Закрыть Sheet → таблица расширяется обратно.
4. Mobile-вьюпорт — Sheet выезжает снизу, не ломает раскладку.
5. Поиск и фильтр цены работают из топ-бара.
