## Что добавляем

В разделе **Авито → Активные объявления** у каждого товара появится новая кнопка **«Сделать N дублей»** (рядом с существующей «Дубли объявления для Авито»). По клику — мини-диалог: ввести число (1–20), опционально шаблон/инструкцию для ИИ, нажать «Запустить». Система автоматически создаст N записей в `avito_listing_variants`, каждая со своим уникальным названием, описанием, перетасованными и заново загруженными фото (новые URL) и небольшим джиттером цены. Дубли сразу попадают в фид Авито (как уже умеет `avito-feed`).

## UX

1. В строке объявления — иконка-кнопка «Размножить» (Layers/CopyPlus).
2. Открывается `BulkDuplicateDialog`:
   - поле «Количество дублей» (1–20, по умолчанию 5),
   - чекбоксы: «Переписать заголовок ИИ», «Переписать описание ИИ», «Перезалить фото (новые URL)», «Сместить цену ±3–10 ₽»,
   - текстовое поле «Инструкция для ИИ» (опц., шаблон стиля),
   - кнопка «Запустить генерацию» с прогресс-баром «3 / 5».
3. По завершении: тост «Создано 5 дублей», список вариантов в `AvitoListingVariantsManager` обновляется. Пользователь может открыть его и при желании подправить.

## Backend: новая edge-функция `avito-bulk-duplicate`

Вход: `{ storeId, sourceProductId, count, options: { rewriteTitle, rewriteDescription, reuploadImages, jitterPrice, instruction } }`.

Шаги:
1. Загрузить исходный товар (`products`) + текущую запись фида (`avito_feed_products`) для baseline title/description/images/price/params.
2. Один вызов VSEGPT (через существующую логику `ai-avito-description`, режим `bulk_uniquify`) — вернуть массив из N пар `{title, description}`. Промпт: «сохрани смысл и характеристики, перефразируй, измени порядок слов/предложений, не повторяйся между вариантами, заголовок ≤ 50 символов, без КАПС, без контактов, seed=i».
3. Для каждого варианта `i = 1..N`:
   - **Фото**: скачать каждый исходный URL → загрузить заново в bucket `avito-images` с путём `duplicates/{storeId}/{productId}/{variantId}/{idx}-{rand}.jpg` → получить новый публичный URL. Дополнительно: лёгкая модификация байтов (1 пиксель/EXIF-stamp) чтобы антифрод по перцептивному хешу тоже спотыкался; на первом этапе достаточно нового пути + новой контрольной суммы (добавим 16 случайных байт в конец JPEG — валидно для JPEG-декодеров). Опционально перетасовать порядок.
   - **Цена**: `price ± rand(3..10)` если включено и цена > 50.
   - **Params**: скопировать `avito_params` исходника, перегенерировать `Id` (артикул) — `ART-{hash}-{i}`, добавить в конец описания, при необходимости.
   - Вставить строку в `avito_listing_variants` со `status='queued'`, `unique_seed=baseSeed+i`, новыми title/description/images/price/avito_params.
4. Вернуть `{ created: N, variantIds: [...] }`.

CORS, валидация zod, верификация владельца магазина через JWT.

## Что уже есть и переиспользуем

- Таблица `avito_listing_variants` + RLS — готово.
- `avito-feed` уже подмешивает варианты в XML как отдельные объявления с уникальными Id, cache-busting фото, jitter цены, daily-seed shuffle — наша задача только наполнить таблицу качественными уникальными вариантами.
- Edge-функция `ai-avito-description` уже умеет режим `uniquify_variant`; расширим её режимом `bulk_uniquify` (вернуть массив) или вызовем N раз — выбираю один батч-вызов (дешевле/быстрее).

## Технические детали анти-дубля

- Новые public URL фото (другой путь в storage, другой ETag).
- Хвост-байты в JPEG (16 байт случайных) → другой md5/SHA, другой pHash-ввод.
- Перетасовка порядка фото с seed = variantSeed.
- Уникальный артикул `ART-XXXX` в конец описания.
- Лёгкая вариация цены.
- AI-перефраз заголовка и описания (разные seed между вариантами).
- Адрес/телефон/категория/contact — **не трогаем** (пользователь явно попросил).

## Лимиты

- Максимум 20 дублей за раз (защита от спама и затрат на ИИ/трафик).
- Прогресс отображается, ошибки по конкретным вариантам не валят весь батч.
