# План: Автоматизация и масс-постинг в Авито

Цель — превратить вкладку «Авито» в полноценный конвейер: массовая загрузка фото с AI-обработкой и уникализацией, двусторонняя синхронизация с Google Таблицами, удобная работа с ошибками и связка с прайс-листом.

---

## Блок 1. Массовые фото + AI пост-обработка и уникализация

**Bulk Photo Manager** — новая панель в `AvitoSection`:

- Drag-and-drop загрузка пачкой (десятки/сотни файлов) с распределением по карточкам по правилам:
  - по имени файла (`SKU_1.jpg`, `SKU_2.jpg`)
  - по выделенным строкам таблицы
  - по «папке» (ссылка на Google Drive — см. Блок 2)
- Очередь обработки с прогрессом, повтор упавших.

**AI-конвейер на каждое фото** (через существующий `avito-image-edit` + Lovable AI / VSEGPT):
- авто-обрезка 4:3, центрирование объекта
- удаление/замена фона (текущая логика)
- легкое улучшение: яркость/контраст/резкость
- **уникализация**: микроповорот ±0.5–2°, шум, легкий кроп, смещение гистограммы, рандом watermark-позиции, рандом сжатия → разные хэши для каждого объявления
- AI upscale для слабых исходников

**Уникализация контента** (описаний/заголовков):
- Кнопка «Уникализировать N» в bulk-панели → для каждой выбранной карточки генерируется новый вариант текста на базе шаблона + параметров товара (Lovable AI). Сохранение версий, чтобы можно было откатить.
- Авто-вставка локальных синонимов, перестановка абзацев, варианты CTA.

**Хранилище**: bucket `avito-images`, метаданные (исходник, варианты, hash) в новой таблице `avito_image_variants`.

## Блок 2. Двусторонняя синхронизация с Google Sheets

**Подключение**: Lovable connector `google_sheets` + `google_drive` (OAuth владельца магазина — отдельный per-user OAuth, т.к. коннектор привязывает аккаунт разработчика; для магазинов делаем кнопку «Подключить Google» с сохранением refresh_token в `store_google_integrations`).

**Структура книги** (создается одной кнопкой «Создать таблицу»):
- `Товары` — основной лист, строка = карточка Авито. Колонки экспортные (зеленые) и внутренние (янтарные), как сейчас в таблице.
- `Ошибки` — выгрузка из отчета Авито, с `Id`-связкой.
- `Фото` — ссылки на папки Google Drive по каждому товару.
- `Группы`, `Лог изменений` — служебные.

**Поток данных (онлайн)**:
- **Из системы → Sheets**: edge function `avito-sheets-sync` пишет батчем при изменении товара/импорте/AI-генерации.
- **Из Sheets → системы**: установка Apps Script триггера `onEdit`, который зовет вебхук edge function `avito-sheets-webhook`. На бэке: дифф по `product_id`, валидация, апдейт `avito_feed_products` + `products` (цена, описание, заголовок, параметры).
- Резервный поллинг раз в N минут на случай отказа webhook.
- Конфликты: «последний выигрывает» + лог в листе `Лог изменений` с автором и timestamp.

**Папка с фото в Drive**:
- В колонке `Фото` лежит ссылка на папку Drive товара.
- Клик по ссылке в Sheets открывает Drive как есть; в нашем UI ссылка из таблицы открывает встроенный **Photo Drawer**: список фото из этой папки (через `google_drive` API), действия: добавить, удалить, переупорядочить (drag), вкл/выкл (флаг «использовать в фиде»), назначить главное. Изменения пишутся и в Drive, и в наш `avito-images`.

## Блок 3. Ошибки Авито в той же таблице

- Кнопка «Загрузить отчет ошибок» (CSV/XLSX от Авито) в шапке вкладки.
- Парсер заполняет таблицу `avito_listing_errors` (product_id, поле, текст ошибки, дата) и одновременно пишет лист `Ошибки` в Google Sheets.
- В основной таблице Авито и в листе `Товары` строки с ошибками подсвечиваются красным; в новой колонке «Ошибки» — счетчик и тултип с описанием.
- Фильтр в сайдбаре «Только с ошибками», bulk-кнопка «AI-исправление» (Lovable AI генерирует исправленные поля → ставит в очередь на апдейт).

## Блок 4. Связь Авито ⇄ Прайс-лист

- На каждой строке прайс-листа: бейдж «В Авито» (по `avito_feed_products`), клик → переход на вкладку Авито с фильтром по этому товару.
- На карточке в Авито: кнопка «Открыть в прайс-листе» → переход в нужный каталог, скролл к товару, открытая фотогалерея.
- Общая фотогалерея: фото товара хранятся в `products.images`; Авито использует подмножество + свои уникализированные варианты. Кнопка «Подтянуть фото из прайса» и «Отправить уникальные обратно в прайс».
- Маршруты: расширить роутинг админки параметрами `?focus=<productId>&from=avito|catalog`.

## Технические детали

**Новые таблицы**
```
store_google_integrations (store_id, refresh_token, spreadsheet_id, drive_root_id)
avito_image_variants (id, store_id, product_id, source_url, variant_url, hash, transforms jsonb)
avito_listing_errors (id, store_id, product_id, field, message, raw, created_at)
avito_sheets_change_log (id, store_id, source, product_id, field, old, new, actor, ts)
```
Все с GRANT + RLS по `store_id` владельца.

**Edge functions**
- `avito-sheets-sync` — push в Sheets (батч, debounce)
- `avito-sheets-webhook` — приём изменений из Apps Script
- `avito-drive-photos` — list/upload/delete/reorder в папке Drive товара
- `avito-bulk-image-process` — очередь AI-обработки/уникализации (вызывает существующий `avito-image-edit` + Lovable AI upscale)
- `avito-errors-import` — парсинг отчета Авито
- `ai-avito-uniquify-text` — генерация уникальных вариантов текста

**Frontend**
- `AvitoBulkPhotoPanel.tsx`, `AvitoPhotoDrawer.tsx` (Drive)
- `AvitoSheetsPanel.tsx` (подключение, статус синка, кнопка «открыть таблицу»)
- `AvitoErrorsBadge.tsx` + подсветка строк в `AvitoSection`
- В `VirtualProductTable` / прайс-листе — бейдж и навигация Авито ⇄ Каталог

**Коннекторы**: `google_sheets`, `google_drive` (на dev-аккаунт для тестов) + per-user OAuth для конечных магазинов (создание OAuth Client ID в Google Cloud, хранение токенов в `store_google_integrations`).

## Порядок реализации

1. Подключить Google коннекторы + сделать минимальный per-user OAuth + таблица `store_google_integrations`.
2. Edge `avito-sheets-sync` + кнопка «Создать таблицу», односторонний экспорт.
3. Apps Script + `avito-sheets-webhook`, двусторонняя синхронизация, лог изменений.
4. Импорт ошибок Авито + лист `Ошибки` + подсветка.
5. Photo Drawer (Drive) + bulk-загрузка фото.
6. AI пост-обработка и уникализация фото (очередь).
7. Уникализация текстов.
8. Связки Авито ⇄ прайс-лист (бейджи, навигация, общие фото).
