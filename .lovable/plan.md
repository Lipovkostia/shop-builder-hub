
## Автоматическое проставление разделов для категорий в прайс-листе

### Что сейчас
В массовом редактировании есть кнопка "Подставить категории" -- она копирует категории товаров из других прайс-листов. Но разделы (папки), в которых эти категории находятся, не копируются. Из-за этого на витрине и в каталоге категории отображаются плоским списком, а не в папках.

### Что будет сделано
При нажатии "Подставить категории" система будет также автоматически копировать настройки разделов (parent_category_id) из исходного прайс-листа в текущий. То есть если в другом прайс-листе категория "Хамон" находится в разделе "Колбаса", то после подстановки эта же структура появится и в текущем прайс-листе.

### Техническая реализация

**Файл: `src/pages/AdminPanel.tsx`**

Расширить обработчик `onBulkAutoFillCategories` (строки 4774-4810):

1. После копирования категорий товаров, собрать все уникальные category ID, которые были скопированы
2. Для каждого такого category ID найти в `catalog_category_settings` исходного прайс-листа запись с `parent_category_id`
3. Если в текущем прайс-листе для этой категории ещё нет записи в `catalog_category_settings` (или `parent_category_id` не задан), создать/обновить запись с тем же `parent_category_id` и `sort_order`
4. Также скопировать записи для самих разделов (parent categories), чтобы они тоже присутствовали в настройках текущего прайс-листа

Логика:
```text
1. Скопировать категории товаров (как сейчас)
2. Собрать все скопированные category IDs
3. Найти catalog_category_settings из исходного каталога для этих категорий
4. Upsert в catalog_category_settings текущего каталога:
   - parent_category_id
   - sort_order
   - custom_name (если есть)
5. Также скопировать настройки разделов-родителей
```

Это один файл, одна функция -- минимальное изменение с максимальным эффектом.
